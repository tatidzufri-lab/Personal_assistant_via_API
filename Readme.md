# Telegram-бот на GPT-4o для генерации контента ВКонтакте

## О проекте

Telegram-бот, который помогает эксперту в нише инфобизнеса быстро создавать контент для ВКонтакте: идеи постов, полноценные публикации, прогревы, продающие тексты, кейсы, отработку возражений и доработку уже написанных материалов.

Бот настраивается под конкретного эксперта — учитывает его тон общения, лексику, целевую аудиторию, продукты и особенности подачи — и генерирует тексты так, будто их написал сам эксперт. По кнопке можно переключать нейросеть: OpenAI GPT-4o или GigaChat.

## Задача и решение

**Проблема:** эксперту нужно регулярно публиковать контент во ВКонтакте — прогревы, продающие посты, кейсы, вовлекающие публикации. Написание каждого поста занимает время, а работа с ChatGPT через браузер требует каждый раз заново объяснять свой стиль, аудиторию и формат соцсети.

**Решение:** бот хранит профиль эксперта и библиотеку готовых сценариев генерации. Достаточно выбрать тип контента кнопкой, ответить на один вопрос — и получить готовый пост, оформленный по правилам ВКонтакте: с цепляющим заголовком, структурой, призывом к действию и хештегами. Результат можно дорабатывать прямо в диалоге («сделай короче», «добавь юмора») или запросить другой вариант одной кнопкой.

## Основной функционал

### Настройка под эксперта

Команда `/setup` запускает анкету из 5 вопросов: имя и ниша, целевая аудитория, тон и лексика, продукты и офферы, особенности подачи и табу. Ответы сохраняются в профиль и автоматически подставляются в системный промпт при каждой генерации — бот выдерживает Tone of Voice эксперта во всех текстах. Профиль можно посмотреть (`/profile`), перенастроить или сбросить.

### Семь сценариев генерации

- 💡 **Идеи для постов** — 10 идей под задачу с заголовками-хуками, описанием и CTA, сгруппированные по целям: прогрев, запуск, продажи, кейсы, возражения, экспертный и вовлекающий контент
- 📝 **Готовый пост** — полноценная публикация: цепляющий заголовок, вступление через боль или интригу, основная часть с конкретикой, вывод, CTA и хештеги
- 🔥 **Прогрев** — пост, который подогревает интерес до продажи без продажи в лоб, с открытой петлёй и идеей следующего шага цепочки
- 💰 **Продающий пост** — текст по продающей структуре: хук → усиление проблемы → решение и выгоды → снятие рисков → оффер → CTA
- 🏆 **Кейс** — оформление фактуры клиента в историю «точка А → что делали → точка Б» с результатом в заголовке
- 🛡 **Отработка возражений** — пост, который разбирает возражение аудитории («дорого», «не получится») через присоединение и переворот
- ✍️ **Доработка текста** — бот улучшает черновик пользователя: усиливает заголовок, чистит структуру, приближает к стилю эксперта и объясняет, что и почему изменил

### Форматирование под ВКонтакте

Все тексты оформляются с учётом особенностей ВК: без markdown-разметки (которую ВК не отображает), короткими абзацами, с умеренными эмодзи, призывами к сохранению, репосту, комментарию или заявке и 3–6 релевантными хештегами. Дополнительно работает программная страховка: если модель всё же вставит markdown, бот вычистит его перед отправкой.

### Переключение нейросети

Кнопка «🧠 Модель» (или команда `/model`) переключает провайдера генерации между OpenAI GPT-4o и GigaChat. Оба провайдера работают через единый интерфейс с одинаковым форматом сообщений, поэтому сценарии и профиль эксперта действуют одинаково для обеих моделей. Если ключи какого-то провайдера не заданы в `.env`, бот сообщит об этом при попытке переключения.

### Диалоговая доработка результата

Бот помнит контекст текущего сценария: после генерации можно обычным сообщением попросить правки («замени CTA на запись в лс», «убери эмодзи») или нажать «🔁 Ещё вариант» — бот перепишет текст с другой подачей. Команда `/reset` очищает контекст.

## Стек технологий

- **Python 3** — основной язык проекта
- **python-telegram-bot** — Telegram-бот: инлайн-кнопки, диалоги (ConversationHandler), обработка команд
- **OpenAI API (Chat Completions, GPT-4o)** — основная модель генерации
- **GigaChat API** — альтернативная российская модель, переключается кнопкой
- **python-dotenv** — безопасное хранение ключей в `.env`
- **logging** — журналирование работы и ошибок

## Структура проекта

- **bot.py** — точка входа: меню, сценарии, обработчики команд и кнопок, диалог настройки профиля
- **prompts.py** — библиотека промптов: системный промпт с правилами оформления под ВК и 7 шаблонов сценариев
- **profiles.py** — профили экспертов: анкета, сохранение и чтение из JSON
- **llm.py** — единый интерфейс к OpenAI и GigaChat, очистка markdown
- **config.py** — загрузка ключей и настроек из `.env`
- **legacy/** — ранние версии проекта (демо работы с Assistant API, LangChain и GigaChat)

## Настройка и запуск

1. Создайте виртуальное окружение и установите зависимости:

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

2. Скопируйте `env-example` в `.env` и заполните:

- `TELEGRAM_TOKEN` — токен бота от @BotFather
- `OPENAI_API_KEY` — ключ OpenAI (модель по умолчанию — gpt-4o)
- `GIGACHAT_CREDENTIALS` — ключ GigaChat (опционально, для переключения модели)

**Важно:** файл `.env` содержит секретные ключи — он добавлен в `.gitignore` и не должен попадать в репозиторий.

3. Запустите бота:

```bash
.venv/bin/python bot.py
```

4. В Telegram отправьте боту `/start`, настройте профиль через `/setup` и выберите сценарий в меню.

Инструкция для пользователя — в файле [INSTRUCTIONS.md](INSTRUCTIONS.md).
